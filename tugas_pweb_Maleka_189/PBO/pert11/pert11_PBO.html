Membuat Sistem Rental Mobil

1. Konsep Utama (Teori OOP)

A. Inheritance (Pewarisan)

  • Konsep: Membuat kelas baru berdasarkan kelas yang sudah ada.
  • Penerapan: Car, Truck, dan LuxuryCar adalah anak dari Vehicle.
  • Manfaat: Tidak perlu tulis ulang variabel model dan baseRate. Cukup tulis di induk, anak otomatis mewarisi.

B. Abstraction (Abstraksi)

  • Konsep: Menyembunyikan detail implementasi.
  • Penerapan: Kelas Vehicle adalah abstract.
  • Alasan: Di dunia nyata tidak ada benda bernama sekadar "Kendaraan". Harus spesifik. Kita melarang pembuatan objek new Vehicle().

C. Overriding (Menimpa)

  • Konsep: Mengubah perilaku metode warisan.
  • Penerapan: Metode calculateRentalCost di Vehicle kosong, tapi di Truck diisi rumus khusus.
  • Manfaat: Setiap anak punya cara hitung sendiri meski nama metodenya sama.

D. Polymorphism (Polimorfisme)

  • Konsep: Satu variabel berubah bentuk jadi banyak jenis objek.
  • Penerapan: Di RentalSystem, variabel bertipe Vehicle bisa berisi Car, Truck, atau LuxuryCar.

2. Bedah Kode (Per File)

File 1: Vehicle.java (Induk)


public abstract class Vehicle {
    protected String model;
    
    public abstract double calculateRentalCost(int days);
}

Penjelasan:

  • abstract class: Menandakan kelas ini tidak bisa dibuat objeknya (hanya cetakan dasar).
  • protected: Agar variabel aman tapi tetap bisa diakses oleh anak kelas (Car/Truck).
  • abstract method: "Metode Kosong" yang memaksa semua anak kelas untuk membuat rumus perhitungan sendiri.

File 2, 3, 4: Kelas Anak (Car, Truck, LuxuryCar)


public class Truck extends Vehicle {
    // ... constructor ...

    @Override
    public double calculateRentalCost(int days) {
        return (baseRate * days) + heavyDutyFee;
    }
}

Poin Penting:

  • extends Vehicle: Kata kunci sakti untuk Pewarisan.
  • super(...): Memanggil constructor orang tua ("Titip simpan data ini").
  • @Override: Penanda kita memodifikasi metode orang tua dengan rumus unik (misal: tambah biaya berat).

File 5: RentalSystem.java (Otak Program)

Bagian 1: Polimorfisme (Persiapan)

Vehicle kendaraanTerpilih = null;

Kita menyiapkan satu variabel kosong bertipe Vehicle. Ini ibarat "Kardus Kosong berlabel Kendaraan".

Bagian 2: Logika Pemilihan


if (pilihan == 1) { 
    kendaraanTerpilih = car; 
} else if (pilihan == 2) { 
    kendaraanTerpilih = truck; 
}

Tergantung input user, "Kardus" tadi akan diisi benda yang berbeda (Mobil atau Truk).

Bagian 3: Eksekusi Ajaib

double total = kendaraanTerpilih.calculateRentalCost(hari);

Saat baris ini jalan, Java mengecek: "Benda apa di dalam kardus?"

  • Jika isinya Truck, rumus Truck yang jalan.
  • Jika isinya Car, rumus Car yang jalan.

3. Alur Jalan Program (Runtime Flow)

  1. Start & Display: Program menyiapkan data mobil di memori dan menampilkan menu 1, 2, 3.
  2. User Input: User memilih "2" (Truk) dan durasi "3 hari".
  3. Process (Polimorfisme): Variabel kendaraanTerpilih diisi objek Truck. Program memanggil rumus Truk: (800rb * 3) + 150rb.
  4. Output: Total biaya Rp 2.550.000 ditampilkan ke layar.
