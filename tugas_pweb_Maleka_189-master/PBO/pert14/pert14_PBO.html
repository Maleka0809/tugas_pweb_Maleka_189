Game Pong

Penjelasan Alur Kerja & Logika Game Pong

Bedah Kode Java Swing (OOP)

1. Konsep Besar: Bagaimana Game Bekerja?

Game ini bekerja menggunakan prinsip animasi "Flipbook" (buku lipat). Komputer melakukan 4 langkah utama secara berulang-ulang dalam kecepatan tinggi (60 kali per detik):

  1. Input: Membaca tombol keyboard yang ditekan user.
  2. Update: Menggeser posisi koordinat (X, Y) bola dan raket.
  3. Cek Aturan: Memeriksa apakah terjadi tabrakan atau bola lewat batas (skor).
  4. Render: Menghapus layar lama dan menggambar ulang posisi baru.

2. Bedah Struktur Kode (Class per Class)

Berikut adalah fungsi spesifik dari setiap file dalam proyek ini:

Nama Class Analogi & Fungsi
PongGame.java
(The Trigger)		 Analogi: Kunci Kontak Mobil.
File ini sangat sederhana, hanya berisi main method. Fungsinya hanya satu: menyalakan GameFrame agar program mulai berjalan.
GameFrame.java
(The Window)		 Analogi: Bingkai Foto.
Class ini membuat jendela aplikasi (tombol close, minimize, title bar). Tugas utamanya adalah menampung (menempelkan) GamePanel di dalamnya.
GamePanel.java
(The Brain)		 Analogi: Kanvas & Wasit.
Ini adalah bagian tersibuk. Tugasnya:
  • Menggambar: Menggunakan Graphics untuk melukis bola dan paddle.
  • Game Loop: Mengatur waktu agar game berjalan stabil di 60 FPS.
  • Collision (Tabrakan): Menentukan kapan bola harus memantul dan kapan reset.
Paddle.java
(The Player)		 Fitur Utama: ID Player.
Objek ini memiliki variabel id.
  • Jika id=1: Hanya merespon tombol W dan S.
  • Jika id=2: Hanya merespon tombol Panah Atas dan Bawah.
Ball.java
(The Moving Object)		 Objek bola yang memiliki logika pantulan dinding sederhana. Bola memiliki method getBounds() yang membuat kotak imajiner untuk mendeteksi tabrakan dengan paddle.

3. Memahami Sistem Koordinat Java

Dalam pemrograman GUI Java (Swing), titik koordinat (0,0) berada di Pojok Kiri Atas layar.

  • Sumbu X (Horizontal): Semakin ke kanan, nilai semakin besar.
  • Sumbu Y (Vertikal): Semakin ke bawah, nilai semakin besar.

Contoh Logika Tabrakan:

// Jika Y bola <= 0, artinya bola menyentuh atap
if (ball.y <= 0) {
    ball.yVelocity = -ball.yVelocity; // Balik arah ke bawah
}

// Jika Y bola >= Tinggi Game, artinya bola menyentuh lantai
if (ball.y >= GAME_HEIGHT - ball.diameter) {
    ball.yVelocity = -ball.yVelocity; // Balik arah ke atas
}

4. Alur Logika "Kalah"

Bagaimana komputer tahu kalau Player kalah?

  1. Setiap detik, komputer mengecek posisi X bola (ball.x).
  2. Jika ball.x < 0 (Negatif/Lewat Kiri) → Player 1 Kalah.
  3. Jika ball.x > Lebar Layar (Lewat Kanan) → Player 2 Kalah.
  4. Jika salah satu kondisi terpenuhi, panggil fungsi newBall() untuk mereset bola ke tengah.
