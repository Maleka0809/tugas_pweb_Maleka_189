Dokumentasi Sistem Manajemen User (CRUD)

Berikut adalah penjelasan detail mengenai cara kerja setiap file dalam sistem manajemen user yang telah dibuat.

1. koneksi.php (Jantung Aplikasi)

File ini adalah jembatan antara kode PHP dan Database MySQL.

  • Fungsi Utama: Menggunakan fungsi mysqli_connect() untuk membuka jalur komunikasi.
  • Parameter: Membutuhkan 4 hal: Host (localhost), User database (root), Password database (kosong), dan Nama Database.
  • Penting: File ini di-include di setiap halaman lain agar tidak perlu menulis ulang kode koneksi berulang kali.

2. index.php (Menampilkan Data / READ)

Halaman utama yang dilihat pengguna.

  • Query: Menggunakan SELECT * FROM users untuk mengambil semua data.
  • Looping (Perulangan): Menggunakan while dan mysqli_fetch_assoc(). Ini artinya: "Selama masih ada data di database, ambil satu baris, tampilkan, lalu ulangi untuk baris berikutnya."
  • Keamanan Tampilan: Menggunakan htmlspecialchars() saat menampilkan nama/username. Ini mencegah hacker menyisipkan skrip berbahaya (XSS) ke dalam nama user.

3. tambah.php (Membuat Data / CREATE)

Formulir untuk memasukkan user baru.

  • Deteksi Tombol: Menggunakan if(isset($_POST['simpan'])) untuk mengecek apakah tombol "Simpan" sudah ditekan.
  • Keamanan Password:
    Penting: Kita menggunakan password_hash($pass, PASSWORD_DEFAULT).
    Fungsi ini mengubah password asli (misal: "rahasia") menjadi kode acak panjang. Jika database bocor, password asli user tetap aman dan tidak bisa dibaca.
  • Query: Menggunakan perintah SQL INSERT INTO.

4. edit.php (Mengubah Data / UPDATE)

Halaman ini sedikit lebih kompleks karena memiliki dua logika untuk password.

  • Mengambil Data Lama: Sebelum form muncul, script mengambil data berdasarkan id di URL agar kolom input terisi data lama.
  • Logika Cerdas Password:
    • Jika kolom password diisi: Update nama, username, DAN password baru (yang di-hash ulang).
    • Jika kolom password kosong: Hanya update nama dan username. Password lama di database tidak diganggu.

5. hapus.php (Menghapus Data / DELETE)

File ini tidak memiliki tampilan (UI), hanya berfungsi memproses perintah.

  • Ambil ID: Menangkap ID dari URL (misal: hapus.php?id=5).
  • Eksekusi: Menjalankan perintah SQL DELETE FROM users WHERE id='...'.
  • Redirect: Setelah dihapus, user langsung "dilempar" kembali ke index.php menggunakan JavaScript window.location.
