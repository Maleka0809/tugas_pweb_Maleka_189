Upload Foto

Dokumen ini menjelaskan bagaimana sistem yang baru saja kita buat bekerja di belakang layar, dari user memilih foto hingga tampil di website.

1. Konsep Paling Penting

Pahamilah perbedaan lokasi penyimpanan:

Kita tidak menyimpan fisik gambar di dalam Database.

  • Folder (Windows Explorer): Tempat menyimpan fisik file gambar (jpg/png).
  • Database (MySQL): Hanya menyimpan TEKS nama filenya saja (contoh: foto_budi.jpg).

2. Alur Perjalanan Data (Step-by-Step)

Bayangkan alurnya seperti mengirim paket. Berikut adalah tahapan yang terjadi:

Tahap A: Di Halaman Form (User)

Saat user mengisi form dan menekan tombol Simpan:

  • Browser membungkus data teks (NIS, Nama) dan file fisik (Foto).
  • Penting: Atribut enctype="multipart/form-data" pada form berfungsi sebagai "amplop khusus". Tanpa kode ini, browser hanya mengirim nama filenya saja, sedangkan fotonya tertinggal di komputer user.

Tahap B: Masuk ke Server (PHP)

Ini adalah bagian tersibuk di file proses_upload.php:

  1. Parkir Sementara: PHP tidak langsung menaruh foto di folder tujuan. Foto "parkir" dulu di folder sementara (Temporary) milik XAMPP dengan nama acak (misal: phpA123.tmp).
  2. Validasi: Script mengecek apakah ada file yang dikirim dan apakah ada error.
  3. Pemindahan (Kunci Utama): Kita menggunakan perintah move_uploaded_file().
    Logikanya: "Ambil file dari parkiran sementara, pindahkan ke folder images/, dan beri nama baru."

Tahap C: Pencatatan (Database)

Setelah file fisik aman tersimpan di folder laptop/server, barulah PHP melapor ke MySQL:

"Halo Database, tolong catat ya: Siswa bernama Budi punya foto dengan nama file 18273_budi.jpg."

Tahap D: Menampilkan Kembali

Saat kita membuka halaman daftar siswa:

  • PHP bertanya ke Database: "Apa nama file fotonya Budi?"
  • Database menjawab: "Namanya 18273_budi.jpg".
  • PHP menyusun HTML: <img src="images/18273_budi.jpg">.
  • Browser menampilkan gambar tersebut yang diambil dari folder.

3. Mengapa Tadi Sempat Error?

Berdasarkan kendala yang dialami sebelumnya, berikut penyebab umumnya:

  • Folder Tidak Ada: PHP (secara default) tidak bisa membuat folder otomatis. Jika folder images belum dibuat manual, PHP bingung mau menaruh file di mana.
  • Izin Akses (Permissions): Kadang folder ada tapi terkunci (Read-only), sehingga PHP dilarang menulis file di situ.
  • Ukuran File: XAMPP memiliki batasan default (biasanya 2MB). Jika foto dari HP ukurannya 5MB, server akan menolak otomatis sebelum kode PHP dijalankan.

Dibuat untuk tujuan pembelajaran logika pemrograman web dasar.
